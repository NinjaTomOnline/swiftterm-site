Support

Help with SwiftTerm

For product questions, setup issues, or bug reports, open a GitHub issue. Do not include passwords, private keys, hostnames, IP addresses, access tokens, or production commands in public reports.

Open a support issue Review privacy policy

Before reporting a connection issue

  • Confirm the host, port, username, and authentication method outside SwiftTerm.
  • For key authentication, verify that the host accepts the public key you assigned in SwiftTerm.
  • Current live SwiftNIO SSH authentication supports password and Ed25519 private keys. RSA keys can be managed and exported, but live RSA auth is still planned.

Files and transfers

  • SFTP browsing supports list, upload, download, delete, rename, and create-directory operations.
  • Local document bookmarks can be reused or removed from the Files screen.
  • Rsync support currently includes planning and queue state. Production rsync-over-SSH execution is a roadmap item.

Subscriptions

SwiftTerm uses Apple In-App Purchase for subscription flows. If a purchase or restore fails in TestFlight or production, include the plan, device model, iOS version, and approximate time of the attempt in your report.

Response path

GitHub issues are the current support channel for pre-release builds. A dedicated support address can be added before App Store launch if the account owner chooses one.
